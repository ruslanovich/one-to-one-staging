Ниже список требований к продукту

Важно: Нужно учитывать, что это поверхностный список, размытый
Важно: Нужно учесть, что начать нужно с тяжелого back-end функционала (обработка входящих файлов, хранилище, ролевая модель)

⸻

Customer Side — экраны и возможности

1) Экран «Логин»

Возможности
	•	Ввести email и пароль.

Переходы
	•	Логин → (после входа) в основной контур приложения (см. ниже: «Список презентаций» / «Отдел»).

⸻

2) Экран «Настройки»

Возможности
	•	[примерно] Загрузить/настроить “систему” / справочники (есть стикер про “загрузить систему … сейлзов”, но часть текста перекрыта).

Переходы
	•	Настройки — отдельный экран из основного меню (на правой мини-схеме он рядом с Логином).

⸻

3) Экран «Загрузка презы»

(на схеме отдельно слева, как часть потока загрузки)

Возможности (часть текста перекрыта)
	•	[примерно] Загрузить файл/транскрипт (текст/аудио/видео).
	•	[примерно] Видеть статус обработки файлов.
	•	[примерно] Связать “сейлза” и “презентацию” (привязка записи к менеджеру).

Переходы
	•	Обычно вызывается из списка презентаций/созвонов как действие “Upload”.

⸻

4) Экран «Отдел»

Возможности
	•	Видеть список сейлзов со средним score и кол-вом презентаций.
	•	Фильтрация по менеджеру / score / кол-ву презентаций.
	•	(Отдельно отмечено) “Возможность фильтрации по параметрам” (общая формулировка).

Переходы
	•	Отдел → Сейлз (выбор конкретного сейлза из списка).

⸻

5) Экран «Сейлз»

Возможности (видимые стикеры)
	•	Видеть имя и фамилию сейлза.
	•	Видеть средний score по сейлзу.
	•	[примерно] Есть блок про “заметку/сводку по презам” (текст размытый, но смысл: видеть/получать краткие выводы по презентациям для этого сейлза).
	•	[примерно] Есть серые стикеры про “динамику/эффективность” (похоже на сравнение/прогресс по периоду и/или сводку по эффективности).

Переходы
	•	Сейлз → Презентация (переход в конкретную запись/презу).
	•	Сейлз → Заметки (переход в список заметок, связанных с этим сейлзом).

⸻

6) Экран «Заметки» (список заметок)

Возможности
	•	Увидеть список заметок для one-to-one.
	•	Фильтровать заметки по дате создания и сейлу/сейлзу.
	•	Просмотреть заметку для one-to-one.
	•	Создавать и редактировать текст заметки в свободной форме [примерно] + “назначать/помечать поле для one-to-one” [примерно] (стикер читается частично).
	•	Фиксация задач в текстовом поле с чекбоксами [примерно] (исполнено/план).

Переходы
	•	Заметки (список) → Заметка (детальная заметка).

⸻

7) Экран «Список презентаций»

Возможности (часть перекрыта)
	•	[примерно] Видеть список презентаций/созвонов с метаданными (дата, менеджер, статус, score — по смыслу).
	•	Фильтрация (видно: по дате загрузки/презы, по менеджеру/score, по параметрам — формулировка частично размыта).
	•	Видеть список тем/обсуждений в презентации в формате drill-down (прямо так написано).

Переходы
	•	Список презентаций → Презентация.

⸻

8) Экран «Презентация»

Возможности (видимые стикеры)
	•	Видеть “разбивку диалога” по чек-листу с приоритетами по блокам [примерно] (формулировка размыта, но смысл именно такой).
	•	Просматривать транскрибацию с тайм-кодами.
	•	“Увидеть проблему со сейлом на месте в разговоре” (то есть привязка проблем/ошибок к конкретным фрагментам).
	•	Видеть потенциал сделки по модели BANT с декомпозицией метрик [примерно].
	•	Есть розовый стикер “Возможность коммент…” — вероятно, комментирование/пометки (текст обрезан).

Что ещё видно на макете справа внизу (пример интерфейса)
	•	Заголовок/ключевой вопрос, обсуждавшийся в созвоне.
	•	Имя файла транскрипта.
	•	Текстовый summary/разбор.
	•	Табличка с блоками оценки (видны столбцы типа): score, rapport, выявление потребностей, презентация продукта / релевантность потребности, отработка возражений, закрытие на следующий шаг (формулировки могут слегка отличаться, но структура такая).

Переходы
	•	Презентация → Заметка (создание/переход к заметке по этому созвону / для one-to-one).

⸻

9) Экран «Заметка» (детальная)

На правой мини-схеме есть отдельный голубой блок «Заметка».

Возможности (по смыслу из связки с “Заметки”)
	•	Просмотр/редактирование заметки.
	•	Чек-лист/задачи (если это часть заметки).
	•	Привязка к сейлзу и к конкретной презентации/созвону.

⸻

10) Экран «Анализ инсайтов» (задел на будущее)

Это серый блок с заметкой:
	•	“данные накапливаются …”
	•	“агрегировать в универсальный продукт”
	•	третий пункт про дальнейшее развитие (текст размытый)

Смысл: отдельный слой/экран для агрегации инсайтов на датасете, чтобы потом сделать “универсальный продукт” поверх накопленных данных.

⸻

Навигация (как на правой мини-схеме)
	•	Логин → (после входа) → Список презентаций
	•	Список презентаций → Презентация → Заметка
	•	Отдел → Сейлз → (Презентация и/или Заметки) → Заметка
	•	Настройки — отдельный экран рядом с Логином в основном меню
